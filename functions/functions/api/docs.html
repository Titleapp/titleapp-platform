TitleApp API Reference

Connect your tools to TitleApp workspaces. Read inventory, push leads, automate tasks, and integrate with any platform.

Base URL https://us-central1-title-app-alpha.cloudfunctions.net/publicApi

Authentication

All API requests require an API key passed in the X-API-Key header. Keys are scoped to specific workspaces and permissions.

Generating a Key

Authenticated users create keys via the createApiKey endpoint using their Firebase Bearer token.

curl -X POST https://us-central1-title-app-alpha.cloudfunctions.net/createApiKey \
  -H "Authorization: Bearer YOUR_FIREBASE_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My Integration",
    "workspace_ids": ["ws_abc123"],
    "scopes": ["read", "write"]
  }'
Using Your Key
curl https://...publicApi/v1/workspaces \
  -H "X-API-Key: ta_your_api_key_here"
Key Properties
FieldDescription
keyThe API key string, prefixed with ta_
workspace_idsArray of workspace IDs this key can access
scopesread, write, or admin
rate_limitRequests per hour (default 100)

Rate Limiting

Rate limits are tracked per API key per hour. Every response includes rate limit headers.

HeaderDescription
X-RateLimit-LimitMaximum requests allowed per hour
X-RateLimit-RemainingRequests remaining in current window
X-RateLimit-ResetUnix timestamp when the window resets
Free tier: 100 requests per hour. Contact us for higher limits.

Error Handling

All errors return a consistent JSON structure.

{
  "error": {
    "code": "not_found",
    "message": "Workspace not found",
    "status": 404
  }
}
StatusCodeMeaning
400bad_requestMissing or invalid parameters
401unauthorizedMissing or invalid API key
403forbiddenKey lacks access to this workspace or scope
404not_foundResource not found
429rate_limitedRate limit exceeded
500internal_errorServer error

Integration

Universal data ingest and webhook subscriptions for real-time events.

POST /v1/workspaces/:workspace_id/inbound Universal data push ▶
Requires write scope. Accepts any JSON payload. Data is queued for AI processing.

Push any structured data into a workspace. The AI will process and route it automatically. Use this as a universal integration point when no specific endpoint exists.

Request Body
FieldTypeDescription
sourcestringoptionalWhere this data came from (default: "unknown")
...anyAny additional JSON fields
Example
curl -X POST .../v1/workspaces/ws_abc123/inbound \
  -H "X-API-Key: ta_..." \
  -H "Content-Type: application/json" \
  -d '{
    "source": "foreflight",
    "type": "flight_log",
    "aircraft": "N12345",
    "hours": 2.5,
    "route": "KORD-KMDW"
  }'
Response
{
  "data": {
    "id": "abc123",
    "status": "pending",
    "message": "Data received and queued for AI processing"
  }
}
